## 2. 接口总览

| 类型 | 名称 | 消息/服务类型 | 方向 | 说明 |
| --- | --- | --- | --- | --- |
| Service | get_robot_mode | crb_ros_msg/srv/GetRobotMode | 订阅 | 查询当前运动模式（简化为 ZERO/STAND/WALK） |
| Service | /set_robot_mode | crb_ros_msg/srv/SetRobotMode | 发布 | 切换 ZERO / STAND / WALK |
| Service | get_robot_state_srv_hl | crb_ros_msg/srv/GetRobotState | 订阅 | 获取运控状态 |
| Service | /motion/upper_body_debug | std_srvs/srv/SetBool | 发布 | 进入/退出上身调试 |
| Service | /motion/whole_body_debug | std_srvs/srv/SetBool | 发布 | 进入/退出全身调试 |
| Service | /motion/nav_upper_body_debug | std_srvs/srv/SetBool | 发布 | 导航模式下启用上身关节控制 |
| Service | /motion/switch_nav_mode | std_srvs/srv/SetBool | 发布 | 导航模式开关（true=NAVIGATION，false=IDLE_MODE） |
| Service | switch_teleoperation | std_srvs/srv/SetBool | 发布 | 进入/退出遥操作行走 |
| Service | switch_autonomous | std_srvs/srv/SetBool | 发布 | 进入/退出动作播放（ACTION_PLAY） |
| Service | imu_calibration | std_srvs/srv/SetBool | 发布 | IMU 校准（实机维护，data=true 触发） |
| Topic | /navigation/cmd_vel | geometry_msgs/msg/Twist | 发布 | 导航/遥操作行走速度指令 |
| Topic | joint_data | sensor_msgs/msg/JointState | 发布 | 遥操作模式上身关节指令 |
| Topic | /upper_body_debug/joint_cmd | crb_ros_msg/msg/UpperJointData | 发布 | 上身调试关节指令 |
| Topic | /motion/debug/joint_cmd | crb_ros_msg/msg/JointStateData | 发布 | 全身调试关节指令（推荐） |
| Topic | /motion/debug/joint_state | crb_ros_msg/msg/JointStateData | 订阅 | 全身调试状态反馈（含 kp/kd） |
| Topic | /motion/joint_cmd | sensor_msgs/msg/JointState | 发布 | 全身关节指令（标准格式） |
| Topic | /motion/joint_state | sensor_msgs/msg/JointState | 订阅 | 全身关节状态（标准格式） |
| Topic | /joint_states | sensor_msgs/msg/JointState | 订阅 | 全身关节实测反馈 |
| Topic | /joint_control | sensor_msgs/msg/JointState | 订阅 | 运控输出指令镜像 |
| Topic | /motion/imu | sensor_msgs/msg/Imu | 订阅 | IMU（仅 WHOLE_BODY_DEBUG 模式发布） |
| Topic | /motion/status | std_msgs/msg/String | 订阅 | 模式/状态摘要 |
| Topic | /motion/robot_state | std_msgs/msg/String | 订阅 | 详细状态字符串 |
| Topic | /motion/current_mode | std_msgs/msg/Int32 | 订阅 | 内部模式枚举值 |
| Topic | /motion/cmd_vel | geometry_msgs/msg/Twist | 订阅 | 运控内部速度镜像 |
| Topic | /imu | sensor_msgs/msg/Imu | 订阅 | 硬件 IMU（实机） |
| Action | basic_action_play | crb_ros_msg/action/BasicActionPlay | 发布 | 预设上肢动作 |

**【说明】** /casbot/event_service 属于上层行为树/SDK 封装，不在 hl_motion 运控节点内。运控层请直接使用 basic_action_play Action 或 switch_autonomous Service。

## 3. 机器人模式说明

### 3.1 内部模式枚举（RobotMode）

| 枚举 | 值 | 说明 |
| --- | --- | --- |
| UNDEFINED | 0 | 上电初始，自动切 READY |
| DAMPING | 1 | 阻尼，SetRobotMode("ZERO") |
| READY | 2 | 站立准备，SetRobotMode("STAND") |
| ACTION_PLAY | 3 | 上身动作播放，switch_autonomous(true) |
| TELEOPERATION | 4 | 遥操作行走，switch_teleoperation(true) |
| IDLE_MODE | 5 | 行走主模式，SetRobotMode("WALK") |
| NAVIGATION | 6 | 导航行走，/motion/switch_nav_mode(true) |
| WHOLE_BODY_DEBUG | 7 | 全身调试，/motion/whole_body_debug(true) |
| UPPER_BODY_DEBUG | 8 | 上身调试，/motion/upper_body_debug(true) |
| WHOLE_BODY_ACTION_PLAY | 9 | 全身动作播放 |
| CALIBRATION | 10 | IMU 校准（内部，imu_calibration 触发） |
| EMERGENCY_STOP | 255 | 急停，所有运动接口冻结 |

### 3.2 get_robot_mode 返回值映射

| 内部模式 | 返回 mode | 返回 mode_name |
| --- | --- | --- |
| UNDEFINED / DAMPING | 2 | "ZERO" |
| READY | 3 | "STAND" |
| 其余所有模式 | 4 | "WALK" |

### 3.3 运控状态（get_robot_state_srv_hl）

| state 值 | 含义 |
| --- | --- |
| 0 | UNDEFINED |
| 1 | DAMPING |
| 2 | READY |
| 3 | STAND |
| 4 | WALKING |
| 5 | RUNNING |
| 255 | EMERGENCY_STOP |

## 4. 使用流程与顺序

| 功能 | 需要的模式 | 进入方式 |
| --- | --- | --- |
| 导航行走 | NAVIGATION | 先 WALK，再 /motion/switch_nav_mode true |
| 遥操作行走 | TELEOPERATION | 先 WALK，再 switch_teleoperation true，下发 /navigation/cmd_vel |
| 遥操作上身控制 | TELEOPERATION | 同上，并向 joint_data 发布 sensor_msgs/JointState |
| 上身关节控制 | UPPER_BODY_DEBUG | /motion/upper_body_debug true |
| 导航中上身控制 | NAVIGATION + 标志位 | /motion/nav_upper_body_debug true |
| 全身关节控制 | WHOLE_BODY_DEBUG | /motion/whole_body_debug true |
| 预设动作 | ACTION_PLAY | 先 STAND，再 switch_autonomous true |

**【安全提示】** 调试模式关节直接跟随指令，首次使用请在扶持/缓冲垫下操作，vel_scale 从 0.05 开始。

### 模式切换状态机（简图）

```
上电 → 自动 READY/STAND
  ├── set_robot_mode("WALK") → IDLE_MODE
  │    ├── switch_nav_mode(true)           → NAVIGATION（/navigation/cmd_vel）
  │    │    └── nav_upper_body_debug(true) → 导航中上身 debug（/upper_body_debug/joint_cmd）
  │    ├── switch_teleoperation(true)      → TELEOPERATION（/navigation/cmd_vel + joint_data）
  │    ├── switch_autonomous(true)         → ACTION_PLAY（basic_action_play）
  │    ├── upper_body_debug(true)          → UPPER_BODY_DEBUG
  │    └── whole_body_debug(true)          → WHOLE_BODY_DEBUG
  └── set_robot_mode("ZERO") → DAMPING → set_robot_mode("STAND") → READY
```

**【提示】** 从 IDLE_MODE 切回 STAND 通常需先切 ZERO 再切 STAND。

## 5. 接口详细说明

### 5.1 获取机器人模式

```
ros2 service call get_robot_mode crb_ros_msg/srv/GetRobotMode "{}"
```

### 5.2 设置机器人模式

```
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'STAND'}"
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'WALK'}"
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'ZERO'}"
```

| mode_name | 目标 | 前提 |
| --- | --- | --- |
| "ZERO" | 阻尼 | 任意 |
| "STAND" | 站立 READY | 通常从 ZERO |
| "WALK" | IDLE_MODE | 从 READY |

### 5.3 获取机器人状态

```
ros2 service call get_robot_state_srv_hl crb_ros_msg/srv/GetRobotState "{start: true}"
```

### 5.4 下肢行走控制（导航模式）

**前提**：NAVIGATION 模式（/motion/switch_nav_mode true）。

| 字段 | 含义 | 备注 |
| --- | --- | --- |
| linear.x | 前进（+）/ 后退（-）m/s | 经内部分段映射，非 1:1 |
| linear.y | 未接入 | 当前无效 |
| angular.z | 左转（+）/ 右转（-）rad/s | 仅 abs(linear.x) >= 0.2 时生效 |

```
# 前进 0.3 m/s
ros2 topic pub /navigation/cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.3, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

# 停止
ros2 topic pub --once /navigation/cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
```

**完整行走流程：**

```
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'STAND'}"
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'WALK'}"
ros2 service call /motion/switch_nav_mode std_srvs/srv/SetBool "{data: true}"
# 发布 /navigation/cmd_vel ...
```

### 5.5 遥操作行走

**前提**：TELEOPERATION 模式（switch_teleoperation true）。

#### 下肢行走

与导航模式相同，向 /navigation/cmd_vel 发布 geometry_msgs/Twist。

#### 上身关节控制

- 话题：joint_data
- 类型：sensor_msgs/msg/JointState（仅含头/臂/腰关节，名称见第 6 节）

进入 TELEOPERATION 后，Teleoperation 模块会先归位到初始姿态，再进入 Running 状态；此后持续向 joint_data 发布关节指令即可驱动上身。

```
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'WALK'}"
ros2 service call switch_teleoperation std_srvs/srv/SetBool "{data: true}"
# 发布 /navigation/cmd_vel ...
# 发布 joint_data（sensor_msgs/JointState）...
ros2 service call switch_teleoperation std_srvs/srv/SetBool "{data: false}"
```

### 5.6 上身关节控制（UPPER_BODY_DEBUG）

#### Step A：进入 / 退出

```
ros2 service call /motion/upper_body_debug std_srvs/srv/SetBool "{data: true}"
ros2 service call /motion/upper_body_debug std_srvs/srv/SetBool "{data: false}"
```

#### Step B：发送关节指令

- 话题：/upper_body_debug/joint_cmd
- 类型：crb_ros_msg/msg/UpperJointData

**消息结构：**

```
std_msgs/Header header
float32 time_ref      # 参考时间(s)，单帧可置 0.1
float32 vel_scale     # 速度缩放 0.0~1.0，首次建议 0.05~0.1
string[]  name        # 关节名（支持带/不带 _joint 后缀，见第 6 节）
float64[] position    # 目标角度 rad
float64[] velocity    # 可选
float64[] effort      # 可选
float64[] kp          # 可选；debug_mode_params.use_default_kp_kd=false 时必填
float64[] kd          # 可选；debug_mode_params.use_default_kp_kd=false 时必填
```

**命令行示例：**

```
ros2 topic pub --once /upper_body_debug/joint_cmd crb_ros_msg/msg/UpperJointData \
  "{header: {stamp: {sec: 0, nanosec: 0}, frame_id: ''},
    time_ref: 0.1, vel_scale: 0.05,
    name: ['head_yaw', 'head_pitch'],
    position: [0.1, -0.1],
    velocity: [], effort: []}"
```

**Python 示例：**

```python
from crb_ros_msg.msg import UpperJointData

msg = UpperJointData()
msg.time_ref = 0.1
msg.vel_scale = 0.1
msg.name = ['left_shoulder_pitch', 'right_shoulder_pitch', 'left_elbow_pitch']
msg.position = [0.3, 0.3, 0.5]
pub.publish(msg)
```

### 5.7 导航中上身控制（nav_upper_body_debug）

在 NAVIGATION 模式下，可通过此 Service 临时允许 /upper_body_debug/joint_cmd 控制头/臂/腰（跳过 RemoteControl 固定姿态复位，使用 debug_mode PD 参数）：

```
ros2 service call /motion/switch_nav_mode std_srvs/srv/SetBool "{data: true}"
ros2 service call /motion/nav_upper_body_debug std_srvs/srv/SetBool "{data: true}"
# 发布 /upper_body_debug/joint_cmd ...
ros2 service call /motion/nav_upper_body_debug std_srvs/srv/SetBool "{data: false}"
```

**【注意】** 仅在 NAVIGATION 模式下可开启；退出导航模式时会自动关闭此标志。旧接口 /motion/nav_head_control 已移除，请统一使用 /motion/nav_upper_body_debug。

### 5.8 全身关节控制（WHOLE_BODY_DEBUG）

#### Step A：进入 / 退出

```
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'WALK'}"
ros2 service call /motion/whole_body_debug std_srvs/srv/SetBool "{data: true}"
ros2 service call /motion/whole_body_debug std_srvs/srv/SetBool "{data: false}"
```

#### Step B：发送关节指令（推荐 JointStateData）

| 项目 | 值 |
| --- | --- |
| 控制话题 | /motion/debug/joint_cmd |
| 反馈话题 | /motion/debug/joint_state |
| 消息类型 | crb_ros_msg/msg/JointStateData |

**消息结构：**

```
std_msgs/Header header
string[]  name
float64[] position
float64[] velocity    # 须与 name 等长
float64[] effort      # 须与 name 等长
float64[] kp          # 见下方 use_default_kp_kd 说明
float64[] kd
```

use_default_kp_kd 为运控配置项（hl_config/motion_params.yaml → debug_mode_params.use_default_kp_kd，默认 true），**不是消息字段**：

- true：可省略 kp/kd，使用 default_pd 默认值
- false：须下发 kp/kd；长度可与 name 等长，或仅覆盖非灵巧手关节（compact layout）

**命令行示例：**

```
ros2 topic pub --once /motion/debug/joint_cmd crb_ros_msg/msg/JointStateData \
  "{header: {stamp: {sec: 0, nanosec: 0}, frame_id: ''},
    name: ['waist_yaw_joint', 'head_pitch_joint'],
    position: [0.3, 0.2],
    velocity: [0.0, 0.0],
    effort: [0.0, 0.0]}"
```

#### Step C：标准 JointState 接口（备选）

| 项目 | 值 |
| --- | --- |
| 控制话题 | /motion/joint_cmd |
| 反馈话题 | /motion/joint_state |
| 消息类型 | sensor_msgs/msg/JointState |

调试模式下若同时收到两种指令，JointStateData（debug 话题）优先。

### 5.9 预设动作播放

#### 方式一：Action（推荐，带反馈）

```
ros2 service call /set_robot_mode crb_ros_msg/srv/SetRobotMode "{mode_name: 'STAND'}"
ros2 service call switch_autonomous std_srvs/srv/SetBool "{data: true}"
ros2 action send_goal basic_action_play crb_ros_msg/action/BasicActionPlay "{type: 'wave_hand'}"
ros2 service call switch_autonomous std_srvs/srv/SetBool "{data: false}"
```

常用 type 值（完整列表见 hl_config/action_data/ 目录及《仿真技能动作部分》第 8 节）：

| type | 说明 |
| --- | --- |
| wave_hand | 挥手 |
| thumb_up | 点赞 |
| heart_gesture | 比心 |
| congratulation_gesture | 恭喜 |
| v_gesture | 剪刀手/耶 |

**Python 示例：**

```python
from rclpy.action import ActionClient
from crb_ros_msg.action import BasicActionPlay

client = ActionClient(node, BasicActionPlay, 'basic_action_play')
client.wait_for_server()
goal = BasicActionPlay.Goal()
goal.type = 'wave_hand'
client.send_goal_async(goal)
```

### 5.10 传感器与状态订阅

```
ros2 topic echo /joint_states          # 实测关节（sensor_msgs/JointState）
ros2 topic echo /joint_control         # 运控输出镜像
ros2 topic echo /motion/imu            # 仅 WHOLE_BODY_DEBUG 有数据
ros2 topic echo /motion/status         # 模式摘要
ros2 topic echo /motion/robot_state    # 详细状态（含 Mode/State/Teleoperation/RemoteControl 等）
ros2 topic hz /joint_states
```

/joint_states 中腿部关节名为 LJ0~LJ5、RJ6~RJ11；手臂等为短名（如 left_shoulder_pitch）。控制指令请使用第 6 节 canonical 名称或别名。

### 5.11 导航模式开关

| 项目 | 值 |
| --- | --- |
| Service | /motion/switch_nav_mode |
| 类型 | std_srvs/srv/SetBool |
| data: true | 切到 NAVIGATION |
| data: false | 回到 IDLE_MODE |
| 前提 | 当前为 IDLE_MODE 且 state 为 STAND，或从 ACTION_PLAY 切回 |

```
ros2 service call /motion/switch_nav_mode std_srvs/srv/SetBool "{data: true}"
ros2 service call /motion/switch_nav_mode std_srvs/srv/SetBool "{data: false}"
```

**【注意】** 此接口与 switch_teleoperation 独立。switch_teleoperation 控制遥操作模式，不再通过 /switch_drive_mode（已废弃）。

### 5.12 IMU 校准（实机维护）

```
ros2 service call imu_calibration std_srvs/srv/SetBool "{data: true}"
```

触发后运控短暂进入 CALIBRATION 模式执行采样，完成后自动恢复原模式。data=false 不执行。

### 5.13 模式总开关汇总

```
# 遥操作
ros2 service call switch_teleoperation std_srvs/srv/SetBool "{data: true}"
ros2 service call switch_teleoperation std_srvs/srv/SetBool "{data: false}"

# 动作播放
ros2 service call switch_autonomous std_srvs/srv/SetBool "{data: true}"
ros2 service call switch_autonomous std_srvs/srv/SetBool "{data: false}"
```

## 7. 上电标准时序

```
1. 机器人上电 → 自动进入 READY（STAND）
2. 确认 get_robot_mode 返回 STAND
3. 延时约 2s 完成姿态归位
4. set_robot_mode("WALK") 进入行走主模式
5. switch_nav_mode(true) 或 switch_teleoperation(true)
6. 下发 /navigation/cmd_vel
7. 关机前 set_robot_mode("ZERO") 卸力
```

## 8. 接口限制与约束

1. linear.y 横向速度未接入，下发无效。
2. 动作播放须先 switch_autonomous(true) 进入 ACTION_PLAY。
3. vel_scale 范围 0~1，超出会被截断。
4. 关节角度存在软件限位（见 hl_config/motion_params.yaml → joint_limits）。
5. 导航指令仅在 NAVIGATION 模式生效；进入前提为 WALK + STAND state。
